# 智能柜物品台账管理系统完成度记录

更新日期：2026-05-08

## 已完成

- 数据库脚本已按 PRD v1.1 字段统一：柜子、格口、称重记录、物品、台账、操作日志。
- 后端基础接口可编译：柜子/格口维护、物品维护、台账查询与导入导出、称重上报、库存盘点、操作日志。
- 开柜接口已改为服务层边界，返回柜号、格口、板地址、锁号、开柜状态，并记录审计日志。
- 称重上报已实现上一条重量比对、默认 5g 阈值、增重/减重事件判定，并自动生成基础台账事件。
- 库存盘点在未传实际重量时会读取各格口最新称重汇总。
- 前端可构建：柜子列表、柜子详情/格口绑定、物品管理、物品台账、库存盘点、称重数据、操作日志。

## 仍需硬件/联调确认

- `cabinet-ledger.hardware-door-control-enabled` 当前默认为 `false`，开柜为模拟结果；真实锁控板 TCP 调用需根据部署方式接入 `skills/open-cabinet/scripts/cabinet-control.mjs` 或改为 Java TCP 客户端。
- 称重采集仍依赖设备端调用 `/api/cabinet/weight/report`，未实现独立 RS485/Modbus/TCP 采集守护进程。
- 自动台账目前只能记录重量事件；要精确关联具体物品，需要开柜操作上下文或用户选择的领用/归还物品。
- 管理员身份认证、HTTPS/TLS、日志防篡改、告警推送还未实现。
