## 1. 项目概述

### 1.1 项目背景
在 isd-robot 项目中，已有一套基于锁控板的智能柜物品领用/归还系统（`open-cabinet` skill）。现需扩展为**带称重模块**的智能柜物品台账管理系统，实现物品存取的自动感知、精确台账记录，以及管理员对物品和柜子的维护能力。

### 1.2 现有系统架构
当前 `open-cabinet` skill 已实现：
- **锁控通信**：TCP 协议连接锁控板（`10.134.231.111:10123`），自定义二进制格式
- **开锁流程**：`开锁 → 监控门状态 → 扣减/归还库存`
- **物品管理**：通过 API 获取物品列表（含柜号、板地址、锁号）

### 1.3 项目目标
- 在现有锁控基础上**集成称重模块**，实现物品存取自动检测
- 建立**物品台账**系统，记录物品全生命周期
- 每个智能柜通过**柜号**关联板地址、锁号、称重模块
- 提供管理员对柜子及物品的维护界面

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| 柜号 | 智能柜的逻辑标识（如 `A01`、`TOOL-01`），用于业务层面标识柜子 |
| 锁控板 | 控制柜门电控锁的硬件板，通过 TCP 通信，一块板可控制多把锁 |
| 板地址 | 锁控板的硬件地址编码（如 `0x01`、`A1`），配置层按字符串保存，用于 TCP 指令寻址 |
| 锁号 | 锁控板上的具体锁编号/编码（如 `01`、`LOCK-01`），对应一个柜门 |
| 称重模块 | 安装于柜体/格口内的重量传感器，用于检测物品重量变化 |
| 物品台账 | 记录物品存取、重量、数量、操作人等信息的电子账目 |
| 管理员 | 具有柜子及物品维护权限的系统用户 |

## 2. 硬件架构

### 2.1 智能柜硬件组成
```
智能柜（柜号标识）
├── 锁控板（1~N 块）
│   ├── 板地址：0x01, 0x02, ...
│   └── 电控锁（1~24 把/板）
│       └── 门磁传感器（检测柜门开关）
├── 称重模块（1~N 个传感器）
│   └── 与柜号/格口绑定
└── 通信模块（WiFi/4G/Ethernet）
```

### 2.2 现有锁控协议（已实现）
- **服务器**：`10.134.231.111:10123`
- **协议格式**：`[0x73 0x74 0x61 0x72] + 命令码 + 板地址 + 锁号 + 数据 + BCC + [0x65 0x6e 0x64 0x6f]`
- **开锁命令**：`0x9a`（单锁）、`0xa0`（全板）
- **状态查询**：通过开锁命令返回门磁状态（`0x00=关`，`0x11=开`）

### 2.3 称重模块接入（待实现）
- 通信方式：待确认（RS485/Modbus / 模拟量 / 独立 TCP）
- 数据格式：重量值（g，精度 1g）
- 采集频率：默认 1 秒/次，可配置
- 变化阈值：默认 5g，可配置

## 3. 功能需求

### 3.1 智能柜管理模块

#### 3.1.1 柜子注册与配置
- **功能描述**：新智能柜接入系统时的注册流程
- **核心功能**：
  - 录入柜号（唯一标识），支持自定义命名
  - 绑定锁控板地址和锁号（一个柜号可对应多把锁/多个格口）
  - 绑定称重模块（传感器 ID、校准参数）
  - 配置柜子基础信息：位置、用途
  - 设置柜子状态：启用/停用/维护中

#### 3.1.2 柜子状态监控
- **功能描述**：实时监控智能柜运行状态
- **核心功能**：
  - 显示柜子在线/离线状态
  - 显示各柜门开/关状态
  - 显示当前柜内总重量（称重模块汇总）
  - 显示柜子空间使用率（已用锁位/总锁位）
  - 异常状态告警：通信中断、硬件故障、重量异常

#### 3.1.3 柜门控制
- **功能描述**：远程控制柜门开关
- **核心功能**：
  - 管理员远程开柜（需身份认证）
  - 用户领用/归还开柜（通过 open-cabinet skill 流程）
  - 开柜原因记录（领用/归还/维护/盘点）
  - 柜门未关超时告警（默认 60 秒）

### 3.2 称重管理模块

#### 3.2.1 称重数据采集
- **功能描述**：实时采集称重模块数据
- **核心功能**：
  - 按柜号/格口定时采集重量数据（默认间隔 1 秒）
  - 重量变化检测（阈值可配置，默认 5g）
  - 原始数据记录与存储
  - 称重模块去皮/校准功能

#### 3.2.2 重量变化分析
- **功能描述**：分析重量变化，推断物品存取行为
- **核心功能**：
  - 检测重量增加 → 推断物品放入
  - 检测重量减少 → 推断物品取出
  - 记录变化量、变化时间戳
  - 与柜门开关事件关联验证
  - 异常变化告警（与台账不符时触发）

#### 3.2.3 称重报表
- **功能描述**：生成称重相关统计报表
- **核心功能**：
  - 按柜号/时间段查询重量变化记录
  - 柜内物品重量分布统计
  - 异常重量变化告警记录

### 3.3 物品台账模块

#### 3.3.1 物品台账记录
- **功能描述**：物品放入/取出时的台账自动记录
- **核心功能**：
  - **自动登记**：检测到重量变化 + 柜门关闭后，自动创建台账记录
  - 物品信息字段：名称、规格、类别、单件重量、数量、存放柜号、存放格口（板地址+锁号）
  - 操作信息：操作人、操作类型（领用/归还）、操作时间
  - 称重关联：记录变化前后重量、变化量

#### 3.3.2 物品信息维护
- **功能描述**：管理物品的基础信息
- **核心功能**：
  - 物品增删改查（名称、规格、类别、标准重量）
  - 物品与柜号/格口绑定关系维护
  - 物品分类管理
  - 批量导入/导出（Excel 模板）

#### 3.3.3 台账查询与报表
- **功能描述**：提供台账数据的查询和统计
- **核心功能**：
  - 按柜号查询当前在库物品清单
  - 按时间段查询存取记录
  - 按物品查询存取历史追溯
  - 生成台账报表（CSV/Excel 导出）
  - **库存盘点**：按格口比对台账重量 vs 实际称重，生成差异报告

### 3.4 管理员维护模块

系统仅设**管理员**单一角色，所有维护功能均由管理员操作。

#### 3.4.1 柜子维护
- **功能描述**：管理员对智能柜进行维护操作
- **核心功能**：
  - 修改柜子基础信息（名称、位置、用途）
  - 锁控板/锁号绑定关系调整
  - 称重模块绑定与重新校准
  - 柜子状态变更（启用/停用/维护中）
  - 柜子故障报修记录

#### 3.4.2 物品维护
- **功能描述**：管理员对台账物品进行维护
- **核心功能**：
  - 批量导入物品台账（模板下载 + Excel 导入）
  - 批量修改物品信息
  - 物品台账数据修正（纠错）
  - 删除/归档历史台账记录
  - 物品分类管理（增删改查物品类别）

#### 3.4.3 操作日志审计
- **功能描述**：记录管理员的所有操作行为，支持审计追溯
- **核心功能**：
  - 记录管理员登录/登出行为
  - 记录柜门开关操作（操作人、时间、原因）
  - 记录物品台账的增删改操作
  - 记录柜子配置变更
  - 支持按时间、管理员、操作类型筛选查询
  - 日志不可删除、不可篡改

## 4. 非功能需求

### 4.1 性能需求
- 称重数据采集延迟 ≤ 1 秒
- 柜门控制指令响应时间 ≤ 2 秒
- 台账查询响应时间 ≤ 3 秒（单柜数据量 ≤ 1000 条）
- 系统支持同时在线管理 ≥ 100 台智能柜

### 4.2 可靠性需求
- 称重数据本地缓存，网络恢复后自动同步
- 柜门控制指令需确认机制，确保执行成功
- 系统故障不影响柜子本地称重和锁控功能

### 4.3 安全需求
- 管理员操作需身份认证
- 操作日志完整记录，不可篡改
- 敏感数据传输加密（HTTPS/TLS）

### 4.4 扩展性需求
- 支持新柜型接入（单柜、多格柜、抽屉柜）
- 支持多称重模块扩展（每格口独立称重）
- API 接口设计遵循 RESTful 规范，便于第三方集成

## 7. 接口设计（关键接口）

### 7.1 柜门控制接口
```
POST /api/cabinet/door/open
Request Body:
{
  "cabinet_id": "A01",       // 柜号
  "slot_id": 1,              // 格口ID（可选，单格口柜子可省略）
  "operator": "admin_id",
  "reason": "领用",           // 领用 | 归还 | 维护 | 盘点
  "auth_token": "xxx"
}
Response:
{
  "code": 200,
  "data": {
    "cabinet_id": "A01",
    "slot_no": "01",
    "door_status": "opened",
    "board_addr": "0x01",
    "lock_number": "05",
    "opened_at": "2026-05-07T10:30:00Z"
  }
}
```

### 7.2 称重数据上报接口（设备端调用）
```
POST /api/cabinet/weight/report
Request Body:
{
  "cabinet_id": "A01",       // 柜号
  "slot_id": 1,
  "sensor_id": "S01",
  "weight": 12.345,
  "timestamp": "2026-05-07T10:30:00Z"
}
Response:
{
  "code": 200,
  "data": {
    "record_id": 10001,
    "change_detected": true,
    "change_amount": 2.150,
    "event_type": 1            // 1=增加 2=减少
  }
}
```

### 7.3 物品台账查询接口
```
GET /api/cabinet/ledger/list?cabinet_id=A01&status=0&category=工具
Query Parameters:
  - cabinet_id: 柜号（必填）
  - status: 状态筛选（可选）
  - category: 类别筛选（可选）
  - page: 页码（可选，默认1）
  - page_size: 每页数量（可选，默认20）
Response:
{
  "code": 200,
  "data": {
    "cabinet_id": "A01",
    "total": 15,
    "page": 1,
    "page_size": 20,
    "items": [
      {
        "id": 1001,
        "item_name": "螺丝刀套装",
        "category": "工具",
        "slot_no": "01",
        "weight": 0.850,
        "quantity": 2,
        "status": 0,
        "stored_at": "2026-05-01T09:00:00Z"
      }
    ]
  }
}
```

### 7.4 库存盘点接口
```
POST /api/cabinet/inventory/check
Request Body:
{
  "cabinet_id": "A01",       // 柜号
  "slot_id": 1,              // 格口ID
  "actual_weight": 15250     // 可选；不传时使用该格口最新称重

Response:
{
  "code": 200,
  "data": {
    "cabinet_id": "A01",
    "slot_id": 1,
    "slot_no": "01",
    "ledger_weight": 15250,       // 该格口台账总重量(g)

    "difference": -70,
    "difference_rate": "-0.46%",
    "status": "normal",           // normal | warning | abnormal
    "checked_at": "2026-05-07T10:30:00Z"
  }
}
```

| 称重模块与锁控板通信冲突 | 数据采集中断 | 独立通信通道，互不干扰 |
| 称重精度偏差 | 台账重量与实际不符 | 增加校准流程，设置合理误差阈值 |
| 网络中断导致数据丢失 | 台账记录不完整 | 设备端本地缓存，恢复后自动同步 |
| 多人同时操作同一柜门 | 数据冲突 | 增加操作锁机制，柜门开启期间禁止远程操作 |

## 10. 附录

### 10.1 柜号命名规则建议
格式：区域-类型-序号
- 示例：`A-TOOL-01` = A区工具柜01号
- 示例：`B-PART-03` = B区零件柜03号
- 支持字母数字组合，长度 4-16 位

### 10.2 称重精度标准
- 常规物品：精度 1g，量程 30000g
- 重型物品：精度 10g，量程 100000g
- 变化检测阈值默认 5g，可配置

### 10.3 格口编号规则
- 单柜单格：直接编号 `01`、`02`...
- 多格柜：按行列编号 `A1`、`A2`、`B1`...
- 与锁号对应关系在配置表中维护
