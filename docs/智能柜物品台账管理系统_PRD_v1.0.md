## 1. 项目概述

### 1.1 项目背景
在 isd-robot 项目中，需要为带称重模块的智能柜开发一套物品台账管理系统。该系统用于管理智能柜内物品的存放、取出、称重记录，并支持管理员对柜子及物品进行维护操作。

### 1.2 项目目标
- 实现智能柜与物品台账的关联管理
- 支持基于称重数据的物品自动识别与记录
- 提供管理员对柜子及物品的全生命周期维护能力
- 确保每个智能柜独立运行，通过唯一ID进行标识和管理

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 智能柜 | 配备称重模块、电控锁具的储物柜单元，具备独立ID |
| 柜号 | 智能柜的唯一标识符（ID），用于系统识别和通信 |
| 物品台账 | 记录柜内物品信息的电子账目，包括物品名称、重量、数量、存取记录等 |
| 称重模块 | 安装于柜体内部的重量传感器，用于检测物品重量变化 |
| 管理员 | 具有柜子及物品维护权限的系统用户 |

## 2. 功能需求

### 2.1 智能柜管理模块

#### 2.1.1 柜子注册与配置
- **功能描述**：新智能柜接入系统时的注册流程
- **核心功能**：
  - 录入柜号（唯一ID），支持自动生成或手动指定
  - 配置柜子基础信息：位置、容量、称重模块参数校准
  - 设置柜子状态：启用/停用/维护中
  - 绑定称重模块硬件地址

#### 2.1.2 柜子状态监控
- **功能描述**：实时监控智能柜运行状态
- **核心功能**：
  - 显示柜子在线/离线状态
  - 显示柜门开/关状态
  - 显示当前柜内总重量
  - 显示柜子空间使用率
  - 异常状态告警（超重、通信中断、硬件故障）

#### 2.1.3 柜门控制
- **功能描述**：远程控制柜门开关
- **核心功能**：
  - 管理员远程开柜（需权限验证）
  - 用户授权开柜（通过身份认证）
  - 开柜原因记录（存物/取物/维护/盘点）
  - 柜门未关超时告警

### 2.2 称重管理模块

#### 2.2.1 称重数据采集
- **功能描述**：实时采集称重模块数据
- **核心功能**：
  - 定时采集重量数据（默认间隔 1 秒，可配置）
  - 重量变化检测阈值设置（默认 5g，可配置）
  - 原始数据记录与存储
  - 称重数据校准与去皮功能

#### 2.2.2 重量变化分析
- **功能描述**：分析重量变化，推断物品存取行为
- **核心功能**：
  - 检测重量增加 → 推断物品存入
  - 检测重量减少 → 推断物品取出
  - 重量变化量记录
  - 变化时间戳记录
  - 与柜门开关事件关联分析

#### 2.2.3 称重报表
- **功能描述**：生成称重相关统计报表
- **核心功能**：
  - 按时间段查询重量变化记录
  - 柜内物品重量分布统计
  - 异常重量变化告警记录

### 2.3 物品台账模块

#### 2.3.1 物品入库登记
- **功能描述**：物品放入智能柜时的台账记录
- **核心功能**：
  - 自动登记：检测到重量增加 + 柜门关闭后，提示确认物品信息
  - 手动登记：管理员主动录入物品信息
  - 物品信息字段：名称、规格、类别、重量、数量、存放人、存放时间、备注
  - 关联称重数据，自动填充重量字段

#### 2.3.2 物品出库登记
- **功能描述**：物品从智能柜取出时的台账记录
- **核心功能**：
  - 自动登记：检测到重量减少 + 柜门关闭后，匹配可能的取出物品
  - 手动登记：管理员主动标记物品取出
  - 记录取出人、取出时间、取出原因
  - 关联称重数据，校验取出重量

#### 2.3.3 物品信息维护
- **功能描述**：管理柜内物品的台账信息
- **核心功能**：
  - 查询柜内物品清单（支持按名称、类别、时间筛选）
  - 编辑物品信息（名称、规格、类别、备注等）
  - 物品位置调整记录
  - 物品状态标记：在库/已取出/待确认

#### 2.3.4 台账查询与报表
- **功能描述**：提供台账数据的查询和统计功能
- **核心功能**：
  - 按柜号查询物品台账
  - 按时间段查询存取记录
  - 物品存取历史追溯
  - 生成台账报表（CSV/Excel 导出）
  - 库存盘点功能：台账重量 vs 实际称重比对

### 2.4 管理员维护模块

系统仅设**管理员**单一角色，所有维护功能均由管理员操作。

#### 2.4.1 柜子维护
- **功能描述**：管理员对智能柜进行维护操作
- **核心功能**：
  - 修改柜子基础信息（位置、容量等）
  - 称重模块重新校准
  - 柜子状态变更（启用/停用/维护中）
  - 柜子故障报修记录

#### 2.4.2 物品维护
- **功能描述**：管理员对台账物品进行维护
- **核心功能**：
  - 批量导入物品台账（模板下载 + Excel 导入）
  - 批量修改物品信息
  - 物品台账数据修正（纠错）
  - 删除/归档历史台账记录
  - 物品分类管理（增删改查物品类别）

#### 2.4.3 操作日志审计
- **功能描述**：记录管理员的所有操作行为，支持审计追溯
- **核心功能**：
  - 记录管理员登录/登出行为
  - 记录柜门开关操作（操作人、时间、原因）
  - 记录物品台账的增删改操作
  - 记录柜子配置变更
  - 支持按时间、管理员、操作类型筛选查询
  - 日志不可删除、不可篡改

## 3. 非功能需求

### 3.1 性能需求
- 称重数据采集延迟 ≤ 1 秒
- 柜门控制指令响应时间 ≤ 2 秒
- 台账查询响应时间 ≤ 3 秒（单柜数据量 ≤ 1000 条）
- 系统支持同时在线管理 ≥ 100 台智能柜

### 3.2 可靠性需求
- 称重数据本地缓存，网络恢复后自动同步
- 柜门控制指令需确认机制，确保执行成功
- 系统故障不影响柜子本地称重和锁控功能

### 3.3 安全需求
- 管理员操作需身份认证
- 操作日志完整记录，不可篡改
- 敏感数据传输加密（HTTPS/TLS）

### 3.4 扩展性需求
- 支持新柜型接入，柜号规则可配置
- 支持多称重模块扩展（多格柜）
- API 接口设计遵循 RESTful 规范，便于第三方集成

## 4. 系统架构

### 4.1 硬件层
```
智能柜硬件
├── 主控板（柜号标识）
├── 称重模块（1~N 个传感器）
├── 电控锁（柜门控制）
├── 门磁传感器（柜门状态检测）
└── 通信模块（WiFi/4G/Ethernet）
```

### 4.2 软件架构
```
物品台账管理系统
├── 设备接入层：智能柜通信协议适配
├── 业务服务层：
│   ├── 柜子管理服务
│   ├── 称重处理服务
│   ├── 台账记录服务
│   └── 权限控制服务
├── 数据存储层：
│   ├── 柜子基础信息表
│   ├── 称重数据表
│   ├── 物品台账表
│   └── 操作日志表
└── 管理界面层：Web 管理后台 / 移动端小程序
```

## 6. 接口设计（关键接口）

### 6.1 柜门控制接口
```
POST /api/cabinet/{cabinet_id}/door
Request:
{
  "action": "open",      // open | close
  "operator": "admin_id",
  "reason": "存物",       // 存物 | 取物 | 维护 | 盘点
  "auth_token": "xxx"
}
Response:
{
  "code": 200,
  "data": {
    "cabinet_id": "CAB001",
    "door_status": "opened",
    "opened_at": "2026-05-07T10:30:00Z"
  }
}
```

### 6.2 称重数据上报接口（设备端调用）
```
POST /api/cabinet/{cabinet_id}/weight
Request:
{
  "weight": 12.345,
  "sensor_id": "S01",
  "timestamp": "2026-05-07T10:30:00Z"
}
Response:
{
  "code": 200,
  "data": {
    "record_id": 10001,
    "change_detected": true,
    "change_amount": 2.150
  }
}
```

```
GET /api/cabinet/{cabinet_id}/items?status=0&category=工具
Response:

    "total": 15,
    "items": [
      {
        "id": 1001,
        "item_name": "螺丝刀套装",

        "weight": 0.850,
```

### 8.2 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 称重精度偏差 | 台账重量与实际不符 | 增加校准流程，设置合理误差阈值 |
| 网络中断导致数据丢失 | 台账记录不完整 | 设备端本地缓存，恢复后自动同步 |
| 多人同时操作同一柜子 | 数据冲突 | 增加操作锁机制，柜门开启期间禁止远程操作 |
| 柜号冲突或重复 | 设备管理混乱 | 柜号生成规则全局唯一校验 |

## 9. 附录

### 9.1 柜号生成规则建议
格式：`{区域码}{类型码}{序号}`
- 示例：`A01-C-001` 表示 A01 区域 C 型柜第 001 号
- 支持字母数字组合，长度 8-16 位

### 9.2 称重精度标准
- 常规物品：精度 1g，量程 30kg
- 重型物品：精度 10g，量程 100kg
- 变化检测阈值默认 5g，可配置
